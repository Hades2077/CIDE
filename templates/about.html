How It Works

1

Upload Files

Upload two code files you want to compare. Supports Python, Java, JavaScript, C/C++, and more.

2

AST Parsing

Code is parsed into Abstract Syntax Trees, extracting structural information independent of naming.

3

Normalization

Variable names, function names, and constants are replaced with generic placeholders.

4

Structural Comparison

Normalized structures are compared to identify identical patterns and logic flow.

5

Weighted Scoring

Combined score calculated: 70% structural similarity + 30% text sequence similarity.

6

Results & Visualization

Detailed results with charts, metrics, and side-by-side diff view.

Detection Capabilities

What We Detect

  • ✅ Renamed variables and functions
  • ✅ Identical code structure
  • ✅ Similar control flow patterns
  • ✅ Matching algorithm implementations
  • ✅ Code with added/removed comments
  • ✅ Reformatted and whitespace changes

What We Ignore

  • ❌ Comment differences
  • ❌ Whitespace and indentation
  • ❌ Variable naming conventions
  • ❌ String literal values
  • ❌ Numeric constant values
  • ❌ Code formatting styles

Plagiarism Types Detected

Exact Copy

Identical code structure (100% structural match). Even with renamed variables.

Renamed Variables

Structural similarity > 90%. Variable/function names changed but logic identical.

Restructured Code

Structural similarity above threshold but < 90%. Code reorganized but similar.
